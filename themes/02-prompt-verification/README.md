# Материалы темы `02-prompt-verification`

## Логика темы
Тема 02 связывает обзорную тему про инструменты ИИ и тему про проектирование образовательных систем.

Задача темы - научить студента получать от LLM не просто «красивый ответ», а **управляемый, проверяемый и безопасный результат**, который затем можно использовать в автоматизации, оценивании и проектировании более сложных решений.

## Структура
- `00_teacher_guide.md` - внутренний сценарий преподавателя с таймингом двух занятий.
- `01_lecture_prompt_design.md` - анатомия сильного промпта, шаблоны и итеративная доработка.
- `02_lecture_verification_and_quality.md` - quality gates, верификация и решение `approve/revise/reject`.
- `03_lecture_safe_use_and_compliance.md` - минимизация ПД, прозрачность использования ИИ и учебный комплаенс.
- `04_lab_prompt_pack.md` - ЛР1: пакет промптов для типовых задач педагога.
- `05_lab_verification_pipeline.md` - ЛР2: контур проверки AI-артефакта перед использованием.
- `06_debrief_rubric.md` - рубрика оценивания по двум лабораторным.
- `images/` - схемы для лекционных материалов.

## Что эта тема не делает
- Не повторяет длинный обзор сервисов из темы 01.
- Не уходит в архитектуру интеграций, pipeline и жизненный цикл ЭСО из темы 03.
- Не требует платных подписок или сложной автоматизации.

## Результаты обучения
К концу темы студент:
- умеет проектировать промпт под конкретную учебную задачу;
- задает ограничения и требуемый формат ответа;
- фиксирует итерации улучшения промпта;
- проводит верификацию AI-результата перед использованием в учебном процессе;
- минимизирует персональные данные и различает допустимое/недопустимое использование ИИ в учебной среде.

## Примечание
Во всех студенческих материалах не указываются временные интервалы. Тайминг хранится только в `00_teacher_guide.md`.

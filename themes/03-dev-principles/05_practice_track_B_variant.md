# 05. Практика - трек B (вариативный): простое образовательное решение по тому же жизненному циклу

## Суть трека B
После основного трека A вы делаете второй, более простой проект, но проходите **все те же фазы жизненного цикла ЭСО**.

> `B-lite` - облегченный вариативный контур. Он обязателен как второй результат, но не дублирует сложность `A-core`.

## Выберите один из 3 вариантов
1. Напоминания о дедлайнах учебных задач.
2. Микро-рекомендации по самообучению после мини-опроса.
3. Опрос удовлетворенности занятием с авто-обратной связью.

## Ограничение на сложность
Проект должен быть простым: минимум компонентов, минимум автоматизаций, но полный жизненный цикл.

## Границы B-lite: допустимо / недопустимо

| Допустимо (acceptable) | Недопустимо (unacceptable) |
|---|---|
| 1 автоматизация + 1 канал уведомления | Многошаговый pipeline уровня трека A с расширенной аналитикой |
| 1-2 простых правила принятия решения | Сложный rule-engine с множественными ветвлениями и подсистемами |
| Минимальная схема данных (6-8 полей) | Избыточная модель данных с неиспользуемыми сущностями |
| Один базовый сценарий проверки + план сопровождения | Полноценный набор нагрузочного/регрессионного тестирования как в A-core |

Если проект выходит за границы `B-lite`, преподаватель возвращает его к упрощению.

## Обязательный результат трека B
1. Короткий lifecycle-canvas (по всем фазам).
2. Минимальная схема данных.
3. Один пример автоматического письма пользователю.
4. Короткая заметка о тестировании и сопровождении (по 2-3 пункта).

## Чек-лист комплаенса РФ для учебного проекта (обязательно заполнить)
Чек-лист применяется как учебно-методическое требование в рамках задания и не заменяет юридическую консультацию.

| Требование | Как подтвердить в работе | Статус |
|---|---|---|
| Целевой характер обработки данных | Показать учебную цель и необходимость каждого поля | `да/нет` |
| Минимизация данных | Оставить только поля, нужные для варианта B | `да/нет` |
| Основание/согласие на обработку | Указать, как в учебной среде обеспечивается правовое основание | `да/нет` |
| Ограничение доступа | Описать, кто имеет доступ к данным и почему | `да/нет` |
| Обезличивание примеров | Использовать тестовые/обезличенные записи | `да/нет` |
| Не передавать лишние ПД в автоматизации | В уведомления и логи включать только необходимый минимум | `да/нет` |

Минимум для зачета `B-lite`: все пункты чек-листа отмечены как `да`.

## Шаблон lifecycle-canvas (заполнить)

| Фаза | Что фиксируем |
|---|---|
| Подготовка | Для кого делаем и какую учебную проблему решаем |
| Идентификация | Кто участвует, какие данные нужны, как поймем успех |
| Концептуализация | Какая логика сервиса и как движутся данные |
| Представление знаний | Какие правила/шаблоны принятия решений используем |
| Программная реализация | Как соберем минимальный рабочий сценарий |
| Тестирование | Как проверим корректность и полезность |
| Сопровождение | Как будем обновлять и улучшать после запуска |

## Минимальная схема данных (шаблон)

| Поле | Тип | Обязательное | Пример |
|---|---|---|---|
| `user_id` | string | да | `stud_1024` |
| `email` | string | да | `student@example.com` |
| `event_type` | string | да | `deadline_reminder` |
| `input_payload` | text/json | да | `{"task":"ЛР-2","due":"2026-03-20"}` |
| `decision` | string | да | `send_reminder` |
| `message_text` | text | да | `Не забудьте сдать ЛР-2 до ...` |
| `status` | string | да | `sent` |

## Шаблон автоматического письма (пример)
```text
Тема: Учебное уведомление: {{event_title}}

Здравствуйте, {{student_name}}!

{{message_text}}

Если нужна помощь, ответьте на это письмо.

С уважением,
Сервис учебной поддержки
```

## Подсказки по каждому варианту

### Вариант 1: Напоминания о дедлайнах
- Вход: дата дедлайна, курс, студент.
- Логика: если до дедлайна осталось мало времени, отправить напоминание.
- Польза: снижение пропусков сроков.

### Вариант 2: Микро-рекомендации по самообучению
- Вход: короткий опрос самооценки и темы, где есть трудности.
- Логика: подобрать 1-2 конкретные рекомендации.
- Польза: персонализация самостоятельной работы.

### Вариант 3: Опрос удовлетворенности занятием
- Вход: ответы на вопросы о понятности, темпе, полезности.
- Логика: сформировать summary для преподавателя и короткую благодарность студенту.
- Польза: быстрая обратная связь и улучшение занятия.

## Критерии зачета по треку B
Зачет `B-lite` ставится, если:
1. Заполнены все фазы жизненного цикла в формате короткого canvas.
2. Схема данных минимальна и соответствует выбранному варианту.
3. Есть рабочий текст автоматического письма.
4. Есть краткая заметка о тестировании и сопровождении.
5. Комплаенс-чеклист РФ заполнен без провалов.

Иначе - доработка `B-lite`.

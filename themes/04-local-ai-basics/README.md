# Материалы темы `04-local-ai-basics`

> Студенту начинать с [01_student_start_here.md](01_student_start_here.md).

## Логика темы
Тема 04 посвящена локальному/offline-контуру ИИ в образовательной практике.

Ее задача - показать будущему преподавателю информатики, что ИИ можно использовать не только через облачные сервисы, но и **на своем устройстве**, если нужно:
- снизить зависимость от подписок и интернета;
- аккуратнее работать с данными;
- показать студентам воспроизводимый локальный маршрут;
- быстро получить базовый учебный результат без сложной инфраструктуры.

Тема рассчитана на **начальный уровень**. Она не уходит в Docker-first инфраструктуру, продвинутый RAG, агентов и GPU-heavy сценарии.

## Структура
### Для студента
- `01_student_start_here.md` - главная точка входа для студента и старт лабораторной работы.
- `02_lecture_why_local_ai.md` - зачем педагогу локальный ИИ-контур и где он оправдан.
- `03_lecture_minimal_local_stack.md` - минимальный набор локальных инструментов для базового курса.
- `04_lecture_safe_local_use.md` - безопасная работа с файлами, моделями и локальным контуром.
- `05_practice_track_A_local_llm.md` - обязательный `A-core`: реальный запуск локальной LLM и создание учебного артефакта.
- `06_practice_track_B_variant.md` - обязательный `B-variant`: локальная транскрибация или локальная работа с документом.
- `07_debrief_rubric.md` - рубрика оценивания по двум контурам.

### Для преподавателя
- `00_teacher_guide.md` - внутренний сценарий преподавателя с таймингом двух занятий.
- `08_teacher_opening_script.md` - teacher-only файл с готовой устной вводной речью и переходом к самостоятельному старту.

### Дополнительно
- `images/` - схемы для лекций и объяснения workflow.

## Как проходит тема
Для студента маршрут такой:
- начать с `01_student_start_here.md`;
- выбрать `Ollama` или `LM Studio` и по умолчанию взять компактную instruct-модель;
- выполнить `05_practice_track_A_local_llm.md`;
- затем выполнить один из вариантов `06_practice_track_B_variant.md`;
- перед сдачей свериться с `07_debrief_rubric.md`.

Teacher-only файлы `00_teacher_guide.md` и `08_teacher_opening_script.md` остаются внутренними материалами преподавателя.

## Что эта тема не делает
- Не дублирует тему 01 как обзор большого числа ИИ-инструментов.
- Не превращается в отдельный курс по prompt engineering.
- Не уходит в проектирование интеграций и жизненный цикл ЭСО из темы 03.
- Не требует обязательного Docker, GPU или self-hosted серверов как порога входа.

## Результаты обучения
К концу темы студент:
- различает локальный и облачный контур использования ИИ;
- может реально запустить локальную LLM на своем устройстве;
- умеет выбрать базовый маршрут `Ollama` или `LM Studio`;
- понимает, как локально использовать транскрибацию или простую работу с документом;
- фиксирует ограничения локального инструмента и не переоценивает его возможности;
- соблюдает базовые правила безопасной работы с файлами и персональными данными.

## Примечание
Во всех студенческих материалах не указываются временные интервалы. Тайминг хранится только в `00_teacher_guide.md`.

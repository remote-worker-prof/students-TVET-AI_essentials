# 01. С чего начать

## В одной фразе
Вы проводите диагностику уровня, создаете мини-модуль по теме `Массивы и циклы` в трех уровнях сложности и подтверждаете качество через сравнение `local` и `ru-cloud`.

## Порядок работы (обязательный)
1. Выполните диагностику уровня:
   - мини-тест,
   - или teacher-форма в `Yandex Forms`,
   - или оба инструмента.
2. Зафиксируйте стартовый уровень по правилам темы.
3. Выполните `Workflow A (Manual Chat)` в [05_practice_track_A_differentiated_module.md](05_practice_track_A_differentiated_module.md).
4. Выполните `B-variant` в [06_practice_track_B_variant.md](06_practice_track_B_variant.md).
5. Проверьте результат по [07_debrief_rubric.md](07_debrief_rubric.md).

## Два workflow
### `Workflow A (Manual Chat)` - обязательно
Вы проверяете и дорабатываете материалы через чаты выбранных LLM.

### `Workflow B (Automation)` - опционально
Вы можете дополнительно выполнить полную автоматизацию на стеке:
`Yandex Forms -> Albato -> GigaChat -> Google Sheets`.

Если вы не выбираете automation, это не снижает возможность зачета при полном `Workflow A`.

## Обязательный стек
- `local`: `Ollama` или `LM Studio`.
- `ru-cloud`: `GigaChat` или `YandexGPT`.

## Что нужно сдать
1. Таблицу диагностики и назначенного уровня.
2. Матрицу дифференциации `уровень -> цель -> задание -> ожидаемый ответ -> критерий`.
3. Журнал итераций (минимум 2 итерации на каждый контур).
4. Сравнение `local vs ru-cloud` по 5 критериям.
5. Решение `approve/revise/reject` по каждому контуру.
6. Финальный комплект материалов для занятия.

## Быстрый старт
1. Прочитайте [02_lecture_differentiation_foundations.md](02_lecture_differentiation_foundations.md).
2. Прочитайте [03_lecture_three_level_design.md](03_lecture_three_level_design.md).
3. Выберите инструменты по [04_lecture_tool_routes_local_vs_ru_cloud.md](04_lecture_tool_routes_local_vs_ru_cloud.md).
4. Начните `A-core` по [05_practice_track_A_differentiated_module.md](05_practice_track_A_differentiated_module.md).

## Что не допускается
- пропускать диагностику уровня;
- использовать только один ИИ-контур;
- заменять таблицы общими описаниями;
- принимать `approve` без evidence.

# 02. Основа темы: единая цель и входная проверка подготовки

## Зачем этот материал
Вы строите задания разной сложности так, чтобы учебная цель оставалась одной и той же. Для этого сначала выполняется входная проверка подготовки (diagnostic level check), а затем выбирается стартовый уровень.

## Единый мини-кейс
Тема всей работы: `Массивы и циклы`.

### Опорный фрагмент (используется без изменений)
1. Массив хранит упорядоченный набор элементов одного типа.
2. Элемент массива выбирается по индексу.
3. Цикл позволяет повторять блок команд.
4. `for` используют, когда известно число повторений.
5. `while` используют, когда повторение зависит от условия.
6. Типичная ошибка: выход за границы массива при неверном индексе.
7. При проходе по массиву важно контролировать начальное и конечное значение индекса.

## Единая учебная цель
Корректно выбирать цикл для прохода по массиву и обнаруживать ошибку индексации.

## Шаг 1. Проведите входную проверку подготовки
Если на этом шаге есть сомнения, используйте правило: сначала минимальная проверка, потом усложнение.

### Что сделать
Выберите один режим:
1. Только мини-тест.
2. Только форма преподавателя (teacher form) в `Yandex Forms`.
3. Оба инструмента.

### Что зафиксировать
Таблицу с источником результата и предварительным уровнем.

| Источник (source) | Сырые данные (raw_result) | Предварительный уровень | Комментарий |
|---|---|---|---|
| мини-тест / форма преподавателя |  |  |  |

### Пример результата
| Источник (source) | Сырые данные (raw_result) | Предварительный уровень | Комментарий |
|---|---|---|---|
| мини-тест | `4 из 6` | `standard` | ошибки в выборе `while` и границ индекса |

## Шаг 2. Выполните мини-тест (если выбран этот инструмент)
### Что сделать
Ответьте на 6 вопросов.

| № | Вопрос | Варианты | Верный ответ |
|---|---|---|---|
| 1 | Что такое массив? | A: набор разных типов; B: упорядоченный набор одного типа; C: цикл | B |
| 2 | Для чего нужен индекс? | A: выбрать элемент; B: завершить программу; C: хранить цикл | A |
| 3 | Когда уместен `for`? | A: когда известно число повторений; B: всегда; C: никогда | A |
| 4 | Когда уместен `while`? | A: когда условие проверяется на каждом шаге; B: только для массивов; C: только для строк | A |
| 5 | Что означает выход за границы массива? | A: индекс вне допустимого диапазона; B: неверный тип; C: нет цикла | A |
| 6 | Что нужно контролировать в цикле? | A: границы индекса; B: цвет текста; C: имя массива | A |

### Что зафиксировать
Количество верных ответов и уровень по шкале.
- `0-2` верных: `basic`.
- `3-4` верных: `standard`.
- `5-6` верных: `advanced`.

### Пример результата
`4 из 6` -> уровень `standard`.

## Шаг 3. Заполните форму преподавателя (если выбран этот инструмент)
### Что сделать
Создайте форму в `Yandex Forms` с 4 полями:
1. `self_confidence` (`1..5`).
2. `last_topic_result` (`низкий/средний/высокий`).
3. `independent_debug` (`да/нет`) — можете ли вы исправить ошибку индексации без подсказки.
4. `pace_preference` (`медленный/средний/быстрый`).

### Что зафиксировать
Ответы формы и уровень по правилу.
- `basic`: `self_confidence <= 2` или `independent_debug = нет`.
- `standard`: `self_confidence = 3` и `last_topic_result = средний`.
- `advanced`: `self_confidence >= 4`, `last_topic_result = высокий`, `independent_debug = да`.

### Пример результата
`self_confidence=2`, `independent_debug=нет` -> уровень `basic`.

## Шаг 4. Назначьте итоговый уровень
Если есть расхождение между двумя источниками, примените единое правило и зафиксируйте решение.

### Что сделать
1. Если источник один, берите его уровень.
2. Если источники совпали, берите совпавший уровень.
3. Если различие на один шаг (`basic/standard` или `standard/advanced`), выбирайте более низкий уровень.
4. Если различие на два шага (`basic/advanced`), назначайте `standard` и добавляйте одну дополнительную проверочную задачу.

### Что зафиксировать
Итоговую строку решения.

| Решение (final_decision) | Итоговый уровень | Обоснование |
|---|---|---|
|  |  |  |

### Пример результата
| Решение (final_decision) | Итоговый уровень | Обоснование |
|---|---|---|
| расхождение на один шаг | `basic` | выбран более безопасный стартовый уровень |

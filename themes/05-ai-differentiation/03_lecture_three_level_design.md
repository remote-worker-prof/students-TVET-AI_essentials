# 03. Проектирование трех уровней: что делать и какой результат получить

## Шаг 1. Зафиксируйте цель (одна для всех уровней)
Цель в этой теме:
`корректно выбирать цикл для прохода по массиву и обнаруживать ошибку индексации`.

## Шаг 2. Зафиксируйте единые критерии
Используйте одинаковые критерии для всех уровней:
- `accuracy` - корректность по опорному фрагменту;
- `goal_fit` - соответствие цели;
- `checkability` - наличие проверяемого ответа;
- `clarity` - понятность формулировки.

## Шаг 3. Разведите сложность, а не тему

| Уровень | Что меняете | Что не меняете |
|---|---|---|
| `basic` | больше подсказок и короткие шаги | цель и критерии |
| `standard` | меньше подсказок, больше самостоятельности | цель и критерии |
| `advanced` | задачи на перенос и анализ ошибок | цель и критерии |

## Полностью заполненный пример мини-модуля

| level | learning_goal | task | expected_answer | criterion |
|---|---|---|---|---|
| basic | корректно выбирать цикл для прохода по массиву и обнаруживать ошибку индексации | Дан массив из 5 элементов. Выберите цикл и укажите диапазон индексов. | Цикл `for`, индексы `0..4`. | Указан корректный тип цикла и корректные границы индекса. |
| standard | корректно выбирать цикл для прохода по массиву и обнаруживать ошибку индексации | Найдите ошибку в проходе: `for i in 0..5` для массива из 5 элементов. | Ошибка: верхняя граница. Должно быть `0..4`. | Ошибка названа и исправлена корректно. |
| advanced | корректно выбирать цикл для прохода по массиву и обнаруживать ошибку индексации | Сравните `for` и `while` для задачи поиска первого отрицательного элемента, объясните риск ошибки индекса. | При известной длине удобен `for`; при условии остановки уместен `while`; риск - выход за границу при неверной проверке индекса. | Есть сравнение, выбор обоснован, риск и способ предотвращения указаны. |

## Шаг 4. Проверьте матрицу по чек-листу
1. Формулировка `learning_goal` одинакова во всех строках?
2. Отличается только сложность задания?
3. Есть проверяемый expected_answer в каждой строке?
4. Критерий проверки конкретен, без абстрактных слов?

## Шаг 5. Подготовьте промпт-каркас
```text
Вы выступаете как методист по информатике.

Тема: массивы и циклы.
Единая цель: корректно выбирать цикл для прохода по массиву и обнаруживать ошибку индексации.
Уровень: [basic/standard/advanced].

Сформируйте:
1) одно задание,
2) ожидаемый ответ,
3) критерий проверки.

Ограничения:
- использовать только заданный опорный фрагмент;
- не добавлять внешние темы;
- сохранить академичный стиль;
- вернуть ответ таблицей.
```

## Что вы получаете на выходе
Выходной артефакт: готовая матрица 3 уровней, которую можно сразу перенести в `A-core`.

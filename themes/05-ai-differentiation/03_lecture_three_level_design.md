# 03. Проектирование трех уровней: единая цель, разная сложность

## Зачем этот материал
Вы учитесь изменять сложность задания, не меняя учебную цель и критерии проверки.

## Шаг 1. Зафиксируйте цель и критерии
### Что сделать
1. Возьмите единую цель темы:
   `корректно выбирать цикл для прохода по массиву и обнаруживать ошибку индексации`.
2. Зафиксируйте единые критерии для всех уровней:
   - корректность содержания (`accuracy`),
   - соответствие цели (`goal_fit`),
   - соответствие уровню (`level_fit`),
   - проверяемость ответа (`checkability`),
   - ясность формулировки (`clarity`).

### Что зафиксировать
Таблицу цели и критериев.

| Элемент | Значение |
|---|---|
| Единая цель | корректно выбирать цикл для прохода по массиву и обнаруживать ошибку индексации |
| Критерии | `accuracy`, `goal_fit`, `level_fit`, `checkability`, `clarity` |

### Пример результата
Цель одна для всех строк матрицы, а критерии повторяются без изменений.

## Шаг 2. Разведите уровни сложности
Если неясно, как различать уровни, используйте простое правило: меняется объем подсказок и глубина рассуждения, но не тема.

### Что сделать
Соберите таблицу различий:

| Уровень | Что добавляете | Что оставляете неизменным |
|---|---|---|
| `basic` | пошаговые подсказки и короткие задания | цель и критерии |
| `standard` | меньше подсказок, больше самостоятельности | цель и критерии |
| `advanced` | перенос в новую ситуацию и разбор ошибок | цель и критерии |

### Что зафиксировать
Краткое правило для каждой строки: чем отличается именно этот уровень.

### Пример результата
`advanced`: добавлен разбор причины ошибки индексации и способ предотвращения ошибки.

## Шаг 3. Сформируйте матрицу мини-модуля
### Что сделать
Заполните матрицу из трех строк.

| Уровень (level) | Цель (learning_goal) | Задание (task) | Ожидаемый ответ (expected_answer) | Критерий (criterion) |
|---|---|---|---|---|
| basic | корректно выбирать цикл для прохода по массиву и обнаруживать ошибку индексации | Дан массив из 5 элементов. Выберите цикл и укажите диапазон индексов. | Цикл `for`, индексы `0..4`. | Тип цикла и границы индекса указаны верно. |
| standard | корректно выбирать цикл для прохода по массиву и обнаруживать ошибку индексации | Найдите ошибку в проходе: `for i in 0..5` для массива из 5 элементов. | Ошибка: верхняя граница. Должно быть `0..4`. | Ошибка названа и исправлена корректно. |
| advanced | корректно выбирать цикл для прохода по массиву и обнаруживать ошибку индексации | Сравните `for` и `while` для поиска первого отрицательного элемента и объясните риск ошибки индекса. | При известной длине удобен `for`; при условии остановки уместен `while`; риск — выход за границу при неверной проверке индекса. | Есть сравнение, обоснование выбора и способ предотвращения ошибки. |

### Что зафиксировать
Готовую матрицу без пустых полей.

### Пример результата
Во всех строках одна цель, но уровень самостоятельности и глубина ответа различаются.

## Шаг 4. Подготовьте каркас запроса к модели
### Что сделать
Используйте единый шаблон запроса:

```text
Вы выступаете как преподаватель информатики.

Тема: массивы и циклы.
Единая цель: корректно выбирать цикл для прохода по массиву и обнаруживать ошибку индексации.
Уровень: [basic/standard/advanced].

Сформируйте:
1) одно задание,
2) ожидаемый ответ,
3) критерий проверки.

Ограничения:
- использовать только опорный фрагмент темы;
- не добавлять внешние темы;
- возвращать ответ в виде таблицы.
```

### Что зафиксировать
Три версии ответа модели: по одной на каждый уровень.

### Пример результата
Для `basic` модель дает короткое задание с подсказкой, для `advanced` — задание с анализом причины ошибки.

## Контрольные вопросы
1. Меняется ли в вашей матрице сложность, а не тема?
2. Есть ли проверяемый ожидаемый ответ в каждой строке?
3. Сможете ли вы объяснить разницу между `standard` и `advanced` в одном предложении?

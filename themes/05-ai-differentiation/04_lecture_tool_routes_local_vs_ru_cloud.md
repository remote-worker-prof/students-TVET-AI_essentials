# 04. Два workflow проверки: manual и automation

## Что выбираете
В теме доступны два workflow:
1. `Workflow A: Manual Chat` - обязательный.
2. `Workflow B: Automation` - опциональный.

## Workflow A: Manual Chat (обязательно)

### Цель
Получить и проверить материалы 3 уровней через чаты LLM без интеграций.

### Шаги
1. Возьмите итоговый диагностический уровень из таблицы `02`.
2. Запустите local-маршрут (`Ollama` или `LM Studio`) и получите версию 1.
3. Запустите ru-cloud маршрут (`GigaChat` или `YandexGPT`) и получите версию 1.
4. Для каждого маршрута сделайте итерацию 2 после исправления дефекта.
5. Заполните сравнение по 5 критериям.
6. Примите `approve/revise/reject`.

### Результат
- заполненный журнал итераций;
- таблица `local vs ru-cloud`;
- финальная версия мини-модуля.

## Workflow B: Automation (опционально, полный контур)

### Стек
`Yandex Forms -> Albato -> GigaChat -> Google Sheets`

### Связь с темой 03
Это тот же интеграционный стек, который уже использовался в теме 03 для автотестирования.
В теме 05 он применяется для автоматизации диагностики и первичного назначения уровня.

### Шаги
1. Создайте диагностическую форму в `Yandex Forms`.
2. В `Albato` настройте `trigger`: новая отправка формы.
3. Передайте ответы в `GigaChat` для вычисления поля `assigned_level` и `level_reason`.
4. Запишите результат в `Google Sheets`.
5. (Опционально) добавьте уведомление о назначенном уровне.

### Обязательный минимум для зачета workflow B
1. Схема потока данных.
2. Таблица маппинга полей.
3. Минимум 1 подтвержденный прогон с записью в Sheets.

### Шаблон маппинга полей

| source | field | target | purpose |
|---|---|---|---|
| Yandex Forms | `self_confidence` | GigaChat prompt | вход диагностики |
| Yandex Forms | `last_topic_result` | GigaChat prompt | вход диагностики |
| Yandex Forms | `independent_debug` | GigaChat prompt | вход диагностики |
| GigaChat | `assigned_level` | Google Sheets | назначенный уровень |
| GigaChat | `level_reason` | Google Sheets | обоснование уровня |

## Какой workflow влияет на зачет
- Полный `Workflow A` достаточен для зачета.
- `Workflow B` засчитывается как расширенный результат.

## Что нельзя делать
- использовать только automation без финального сравнения local/ru-cloud;
- пропускать ручную проверку результата;
- принимать итог без таблицы evidence.

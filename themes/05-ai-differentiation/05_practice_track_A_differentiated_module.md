# 05. Практика - трек A (`A-core`): диагностика + 3 уровня + сравнение двух контуров

## Формат
Индивидуальная работа.

`A-core` обязателен полностью.

## Что вы делаете
1. Выполняете диагностику уровня.
2. Назначаете стартовый уровень.
3. Формируете материалы трех уровней в двух контурах.
4. Сравниваете качество `local` и `ru-cloud`.
5. Принимаете `approve/revise/reject`.

## Обязательные входные данные

### Опорный фрагмент
1. Массив хранит упорядоченный набор элементов одного типа.
2. Элемент массива выбирается по индексу.
3. Цикл позволяет повторять блок команд.
4. `for` используют, когда известно число повторений.
5. `while` используют, когда повторение зависит от условия.
6. Типичная ошибка - выход за границы массива при неверном индексе.
7. При проходе по массиву важно контролировать начальное и конечное значение индекса.

### Единая цель
Корректно выбирать цикл для прохода по массиву и обнаруживать ошибку индексации.

### Единые критерии
- `accuracy`
- `goal_fit`
- `level_fit`
- `checkability`
- `clarity`

## Обязательные инструменты
- `local`: `Ollama` или `LM Studio`.
- `ru-cloud`: `GigaChat` или `YandexGPT`.

# 07. Рубрика оценивания и формат защиты

## Рубрика (0-2 балла)

| Критерий | 0 | 1 | 2 |
|---|---|---|---|
| Диагностический вход | нет диагностики | диагностика неполная | диагностика полная и уровень обоснован |
| Единая цель | не зафиксирована | зафиксирована частично | едина и явно сохранена в 3 уровнях |
| Трехуровневая структура | уровни не различаются | различия частичные | различия корректны и проверяемы |
| Конкретность формулировок | абстрактно | частично конкретно | шаги, артефакты и ожидаемые результаты указаны явно |
| Два ИИ-контура | использован один | второй подтвержден частично | оба контура подтверждены полностью |
| Сравнение 5 критериев | отсутствует | частично заполнено | заполнено с evidence |
| Решения по качеству | отсутствуют | есть без обоснования | есть по обоим контурам и обоснованы |
| Корректность workflow | workflow не подтвержден | workflow подтвержден частично | выбранный workflow воспроизводим и подтвержден |

## Порог зачета
- Рекомендуемый минимум: `12` из `16`.
- Обязательные условия:
  1. Диагностика не ниже `2`.
  2. Два ИИ-контура не ниже `2`.

## Правило workflow
- Полный `Workflow A (Manual Chat)` достаточен для зачета.
- `Workflow B (Automation)` учитывается как расширенный результат.

## Обязательные артефакты
1. Диагностическая таблица.
2. Матрица 3 уровней.
3. Журнал итераций.
4. Сравнение `local vs ru-cloud`.
5. Решение `approve/revise/reject`.
6. Финальный комплект материалов.

## Чек-лист быстрой проверки

| Пункт | Статус |
|---|---|
| Диагностика выполнена и обоснована | `да/нет` |
| Единая цель сохранена во всех уровнях | `да/нет` |
| Использованы два ИИ-контура | `да/нет` |
| Сравнение по 5 критериям заполнено | `да/нет` |
| Решения по качеству подтверждены evidence | `да/нет` |
| Ограничения маршрутов зафиксированы | `да/нет` |

## Формат защиты
Во время защиты покажите:
1. Как выполнена диагностика и назначен уровень.
2. Чем отличаются local и ru-cloud результаты.
3. Почему принято `approve/revise/reject`.
4. Какие ограничения зафиксированы.

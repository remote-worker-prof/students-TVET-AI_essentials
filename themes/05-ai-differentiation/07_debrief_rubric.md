# 07. Рубрика оценивания и формат защиты

## Как читать рубрику
Каждый критерий оценивается по шкале `0-2`:
- `0` — критерий не выполнен,
- `1` — выполнен частично,
- `2` — выполнен полностью и подтвержден.

## Рубрика (0-2 балла)

| Критерий | 0 | 1 | 2 |
|---|---|---|---|
| Входная проверка подготовки | нет проверки | проверка неполная | проверка полная, уровень обоснован |
| Единая учебная цель | цель не зафиксирована | цель зафиксирована частично | цель сохранена во всех 3 уровнях |
| Логика трех уровней | уровни не различаются | различия частичные | различия четкие и проверяемые |
| Конкретность формулировок | общие слова без действий | есть часть конкретики | есть шаг, артефакт и пример результата |
| Два контура моделей | использован один контур | второй контур подтвержден частично | оба контура подтверждены полностью |
| Сравнение по 5 критериям | отсутствует | заполнено частично | заполнено полностью с доказательствами |
| Решение по качеству | нет решения | решение без обоснования | решение обосновано для каждого контура |
| Корректность порядка работы (workflow) | порядок не подтвержден | подтвержден частично | выбранный порядок воспроизводим |

## Порог зачета
- Рекомендуемый минимум: `12` из `16`.
- Обязательные условия:
  1. «Входная проверка подготовки» = `2`.
  2. «Два контура моделей» = `2`.

## Правило по порядкам работы
- Полный базовый порядок проверки в чате (Manual Chat Workflow) достаточен для зачета.
- Расширенный автоматизированный порядок проверки (Automation Workflow) засчитывается как усиление практики.

## Обязательные артефакты
1. Таблица входной проверки и итогового уровня.
2. Матрица трех уровней.
3. Журнал итераций.
4. Сравнение двух контуров по пяти критериям.
5. Решение по качеству: принять / доработать / не принимать.
6. Финальный комплект материалов для занятия.

## Чек-лист быстрой проверки

| Пункт | Статус |
|---|---|
| Входная проверка выполнена и обоснована | `да/нет` |
| Единая цель сохранена во всех уровнях | `да/нет` |
| Использованы два контура моделей | `да/нет` |
| Сравнение по пяти критериям заполнено | `да/нет` |
| Решения по качеству подтверждены | `да/нет` |
| Ограничения контуров зафиксированы | `да/нет` |

## Формат защиты
При защите покажите:
1. Как выполнена входная проверка и назначен итоговый уровень.
2. Чем различаются результаты двух контуров.
3. Почему принято каждое решение по качеству.
4. Какие ограничения и условия применения вы зафиксировали.

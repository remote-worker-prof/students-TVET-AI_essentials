# 08. Вводная речь для старта темы (внутренний файл)

## Короткий сценарий открытия (2-4 минуты)

Сегодня вы запускаете тему, где нужно сделать не «красивый текст от ИИ», а проверяемый учебный результат.

Порядок работы строгий:
1. Сначала диагностика уровня.
2. Затем сборка материалов трех уровней по одной цели.
3. Затем сравнение local и ru-cloud.
4. Затем решение `approve/revise/reject` по evidence.

В теме есть два workflow:
- `Workflow A (Manual Chat)` - обязателен.
- `Workflow B (Automation)` - опционален, на стеке темы 03:
  `Yandex Forms -> Albato -> GigaChat -> Google Sheets`.

Для зачета достаточно полного `Workflow A`.
Если вы выбираете automation, это засчитывается как расширенный результат.

Итог темы: готовый комплект материалов, который можно сразу применять на занятии.

## Переход к действиям
1. Откройте [01_student_start_here.md](01_student_start_here.md).
2. Выполните диагностику и назначьте стартовый уровень.
3. Перейдите в [05_practice_track_A_differentiated_module.md](05_practice_track_A_differentiated_module.md).
4. После `A-core` выполните один вариант из [06_practice_track_B_variant.md](06_practice_track_B_variant.md).

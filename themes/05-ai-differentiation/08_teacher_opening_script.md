# 08. Вводный сценарий запуска темы (внутренний файл)

## Текст открытия (2-4 минуты)
Сегодня вы запускаете тему, в которой важен не «красивый текст», а проверяемый учебный результат.

Порядок действий:
1. Сначала выполняется входная проверка подготовки.
2. Затем собираются задания трех уровней по одной цели.
3. После этого сравниваются два контура моделей.
4. В конце принимается решение по качеству: принять / доработать / не принимать.

В работе используются два порядка:
- базовый порядок проверки в чате (Manual Chat Workflow) — обязателен;
- расширенный автоматизированный порядок проверки (Automation Workflow) — опционален, на связке `Yandex Forms -> Albato -> GigaChat -> Google Sheets`.

Для зачета достаточно полного базового порядка проверки в чате.
Если выбран автоматизированный порядок, это засчитывается как расширенный практический результат.

Итог темы: комплект материалов, который можно сразу применять на занятии.

## Переход к действиям
1. Откройте [01_student_start_here.md](01_student_start_here.md).
2. Выполните входную проверку и назначьте стартовый уровень.
3. Перейдите в [05_practice_track_A_differentiated_module.md](05_practice_track_A_differentiated_module.md).
4. После обязательной части A выберите один вариант в [06_practice_track_B_variant.md](06_practice_track_B_variant.md).

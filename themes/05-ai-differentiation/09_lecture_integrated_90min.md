# 09. Цельная лекция: дифференциация с ИИ (расширенный маршрут)

---

## 🎯 Цель лекции
Вы проходите полный маршрут проектирования дифференцированного учебного модуля по теме `Массивы и циклы`: от входной проверки подготовки до выбора рабочего контура модели и обоснованного решения по качеству.

Итог работы: у вас есть рабочий алгоритм, по которому можно разработать и проверить три полных комплекта материалов уровней `basic`, `standard`, `advanced`, а затем перейти к обязательной части A и вариативной части B без потери качества и без дублирования действий.

Дифференциация в этом материале рассматривается как управляемое изменение сложности и формата работы при сохранении общей учебной цели [1], [2].

---

## 📚 Словарь ключевых терминов
- **Входная проверка подготовки (diagnostic level check)**: короткая проверка до начала основной работы, которая помогает назначить стартовый уровень.
- **Порядок работы (workflow)**: последовательность действий от постановки цели до решения по качеству.
- **Базовый порядок проверки в чате (Manual Chat Workflow)**: создание и доработка материалов через чаты языковых моделей.
- **Расширенный автоматизированный порядок проверки (Automation Workflow)**: связка `Yandex Forms -> Albato -> GigaChat -> Google Sheets`.
- **Содержание (content)**: что именно вы объясняете и какими примерами.
- **Процесс работы (process)**: как организована последовательность действий.
- **Учебный продукт (product)**: что считается итоговым доказуемым результатом.
- **Учебная среда (learning environment)**: условия, темп, поддержка и ограничения.
- **Правильные ответы теста**: эталон ответов для проверки тестовой части.
- **Проходной балл (минимум правильных ответов)**: порог, после которого результат считается достаточным.
- **Ошибка для разбора (в учебном примере)**: заранее включенная в материал ошибка с пояснением и образцом корректного исправления; это не статистика чьих-либо реальных ошибок.
- **Решение по качеству**: `принять / доработать / не принимать (approve / revise / reject)`.

Если термин употребляется впервые, в тексте сохраняется формат: русский термин + английский эквивалент в скобках.

## 🧭 Маршрут лекции по времени
1. Блок 1 (10-12 минут): рамка, цель, критерии качества и логика артефактов.
2. Блок 2 (15 минут): входная проверка подготовки и назначение стартового уровня.
3. Блок 3 (20 минут): дифференциация через `content/process/product/learning environment`.
4. Блок 4 (20 минут): проектирование трех полных комплектов (`basic`, `standard`, `advanced`).
5. Блок 5 (15 минут): сравнение локального и российского облачного контуров, решение по качеству.
6. Блок 6 (8-10 минут): переход к части A/B, типовые риски и самопроверка.

Каждый блок выполнен в одном формате:
- `🎯 Зачем`
- `🧱 Что сделать`
- `📑 Что зафиксировать`
- `✅ Пример результата`
- `⚠️ Риск шага`
- `🔧 Как исправить`

## 🧱 Блок 1. Рамка, цель и критерии качества
### 🎯 Зачем
Сначала нужно зафиксировать границы работы, иначе дифференциация быстро превращается в набор несвязанных фрагментов.

Прозрачная постановка задания через явные пункты «зачем делается», «что именно делать» и «по каким критериям проверяется» повышает воспроизводимость результата [3].

### 🧱 Что сделать
1. Зафиксируйте **единую учебную цель** для всех уровней.
2. Зафиксируйте, какие элементы могут изменяться между уровнями, а какие должны оставаться неизменными.
3. Зафиксируйте минимально обязательный набор артефактов:
   - Таблица 1: стартовый уровень + матрица уровней;
   - Таблица 2: сравнение контуров + решение по качеству;
   - Таблица 3: полный комплект `basic`;
   - Таблица 4: полный комплект `standard`;
   - Таблица 5: полный комплект `advanced`;
   - финальный комплект материалов;
   - результат вариативной части B.
4. Зафиксируйте контрольный вопрос на целостность:
   - «Можно ли проверить достижение цели по наблюдаемым признакам, а не по впечатлению?»
5. Зафиксируйте правило прозрачности задания (Purpose/Task/Criteria):
   - Purpose: зачем нужен блок;
   - Task: что конкретно делать;
   - Criteria: по чему проверять качество [3].

### 📑 Что зафиксировать
Короткую стартовую запись.

| Поле | Что записать |
|---|---|
| Единая цель | Одна формулировка, общая для всех уровней |
| Изменяемые элементы | Что именно меняется между уровнями |
| Неизменяемые элементы | Что остается одинаковым |
| Обязательные артефакты | Полный список сдачи |
| Критерий проверяемости | Один проверочный вопрос |

### ✅ Пример результата
| Поле | Пример заполнения |
|---|---|
| Единая цель | `Выбирать корректный цикл для задачи с массивом и обнаруживать ошибку индексации` |
| Изменяемые элементы | `Степень самостоятельности, объем объяснения, сложность задач` |
| Неизменяемые элементы | `Тема, целевая операция, критерии точности ответа` |
| Обязательные артефакты | `Т1-Т5 + финальный комплект + результат B` |
| Критерий проверяемости | `Решение содержит выбор цикла, объяснение и проверку индекса` |

### ⚠️ Риск шага
Формулировка цели слишком общая: «изучить тему», «понять материал», «разобраться лучше».

### 🔧 Как исправить
Примените тест формулировки цели:
1. В цели есть глагол наблюдаемого действия.
2. В цели есть объект действия.
3. В цели есть признак корректности.

Если один из пунктов отсутствует, цель нужно уточнить.

## 🧱 Блок 2. Входная проверка подготовки и стартовый уровень
### 🎯 Зачем
Стартовый уровень должен назначаться по данным, а не по интуитивной оценке. Формирующее оценивание (formative assessment) дает наилучший эффект, когда обратная связь своевременна и привязана к конкретному действию [5], [6].

### 🧱 Что сделать
#### Шаг 1. Подготовьте мини-тест входной проверки
Сделайте 6 вопросов по теме `Массивы и циклы`.

Правило интерпретации:
- `0-2` правильных ответа -> `basic`
- `3-4` правильных ответа -> `standard`
- `5-6` правильных ответа -> `advanced`

Пример мини-теста:
1. Что означает индекс `0` в массиве?
2. Когда для прохода по массиву целесообразно использовать `for`?
3. Что происходит при обращении к индексу `length`?
4. Какой цикл удобнее, если заранее известна длина массива?
5. Найдите ошибку в условии `i <= arr.length`.
6. Что изменится, если заменить `i++` на `i += 2`?

#### Шаг 2. Подготовьте форму преподавателя (teacher form) в Yandex Forms
Добавьте поля:
- краткий уровень опоры по базовым конструкциям;
- устойчивость при выполнении пошаговой инструкции;
- способность объяснить выбор цикла;
- способность найти ошибку индексации;
- итоговая оценка уровня (`basic / standard / advanced`).

Рекомендуемый тип полей:
- шкала 1-3,
- короткий текст для комментария,
- обязательный итоговый селектор уровня.

#### Шаг 3. Примените алгоритм назначения уровня
1. Если есть только один источник (только мини-тест или только форма), используйте его результат.
2. Если два источника совпали, используйте совпавший уровень.
3. Если расхождение в один шаг (`basic` vs `standard` или `standard` vs `advanced`), выбирайте более низкий уровень.
4. Если расхождение в два шага (`basic` vs `advanced`), назначайте `standard` и добавляйте уточняющую задачу.
5. После уточняющей задачи зафиксируйте окончательный уровень.

#### Шаг 4. Подготовьте уточняющую задачу для случая сильного расхождения
Формат:
- задание на выбор цикла,
- задание на объяснение выбора,
- задание на обнаружение ошибки индексации.

Пример уточняющей задачи:
«Дан массив оценок. Выведите все значения выше 70, затем объясните, почему выбран именно этот цикл, и укажите, где может возникнуть выход за границу массива.»

3. С какого уровня начинается работа в этой теме.

2. Для каждого расхождения фиксируйте примененное правило.
3. Обратную связь всегда давайте в формате «сильная сторона -> зона доработки -> стартовый уровень».

## 🧱 Блок 4. Проектирование трех полных комплектов
### 🎯 Зачем
В этом блоке вы превращаете рамку и матрицу уровней в полноценные материалы, которые можно использовать сразу. Полный комплект для каждого уровня нужен, чтобы результат был проверяем, повторяем и пригоден к практическому применению.

### 🧱 Что сделать
#### Шаг 1. Примените единый каркас комплекта для каждого уровня
Каждый комплект (`basic`, `standard`, `advanced`) должен включать:
1. Теорию: краткое объяснение, пример, опорные понятия.
2. Практику: 3 задания, ожидаемые ответы, критерии.
3. Тест: 6 вопросов.
4. Правильные ответы теста.
5. Проходной балл (минимум правильных ответов).
6. Ошибку для разбора (в учебном примере).
7. Образец корректного исправления.

#### Шаг 2. Сформируйте комплект `basic`
##### Теория `basic`
Цель теории: дать устойчивый базовый алгоритм прохода по массиву.

Опорный текст:
- Массив хранит элементы в последовательности с индексами, начиная с `0`.
- Если нужно пройти все элементы, чаще всего подходит цикл `for` с шагом `1`.
- В условии цикла верхняя граница не включается.

Мини-пример:
```text
for (i = 0; i < arr.length; i++)
```

Опорные понятия:
- индекс;
- длина массива;
- условие цикла;
- шаг цикла.

##### Практика `basic` (3 задания)
Задание 1:
- Условие: «Пройдите массив `scores = [78, 91, 66]` и выведите каждый элемент».
- Ожидаемый ответ: цикл с индексом от `0` до `< length`.
- Критерий: нет выхода за границу массива.

Задание 2:
- Условие: «Найдите ошибку в условии `i <= scores.length`».
- Ожидаемый ответ: заменить `<=` на `<`.
- Критерий: объяснено, почему индекс `length` недопустим.

Задание 3:
- Условие: «Измените цикл, чтобы выводить только элементы с четными индексами».
- Ожидаемый ответ: шаг `i += 2` при старте с `0`.
- Критерий: правильно описано влияние шага на выбор элементов.

##### Тест `basic` (6 вопросов)
1. С какого индекса начинается массив?
2. Какое условие корректно для прохода по всем элементам?
3. Что означает `arr.length`?
4. Почему `i <= arr.length` ошибочно?
5. Что изменяет `i++`?
6. Как пройти элементы с четными индексами?

Правильные ответы теста `basic`:
1. `0`
2. `i < arr.length`
3. Количество элементов
4. Индекс `length` выходит за границу
5. Увеличивает индекс на 1
6. Шаг `2` при старте с `0`

Проходной балл `basic`:
- минимум `4/6`.

Ошибка для разбора `basic`:

- вариант 2: углубление и перенос;
- вариант 3: автоматизация проверки (опционально).

Правило:
- часть B не дублирует таблицы части A;
- часть B показывает направленную доработку.

#### Шаг 3. Проведите контроль «разработка материала vs отчетность»

- `🧱 Разработка материала`: создание учебного контента.

#### Шаг 4. Выполните анти-смешение ролей
Проверьте каждый пункт инструкции:
- если в пункте одновременно присутствуют глаголы «создать» и «записать в отчет», разделите его на два подпункта.

## 🧪 Контрольный раздел: 12 типовых срывов выполнения и их коррекция
### 1. Цель сформулирована как «разобраться в теме»
- Что не так: цель непроверяема.
- Как исправить: добавить наблюдаемое действие и признак корректности.

### 2. Входная проверка есть, но не влияет на уровень
- Что не так: проверка становится формальной.
- Как исправить: использовать явный алгоритм назначения уровня.

### 3. Уровни отличаются только длиной текста
- Что не так: фактическая сложность не растет.
- Как исправить: фиксировать разные операции мышления.

### 4. В `advanced` нет анализа причины ошибки
- Что не так: уровень не достигает цели переноса и причинного разбора.
- Как исправить: добавить отдельное задание «ошибка -> причина -> устойчивое исправление».

### 5. В практике нет ожидаемых ответов
- Что не так: проверка становится субъективной.
- Как исправить: добавить ожидаемый ответ к каждому заданию.

### 6. В практике нет критериев
- Что не так: нельзя провести единообразную оценку.
- Как исправить: добавить минимум один наблюдаемый критерий.

### 7. В тесте нет правильных ответов теста
- Что не так: невозможно быстро проверить корректность.
- Как исправить: добавить полный список правильных ответов.

### 8. Нет проходного балла
- Что не так: не определен минимум приемлемого результата.
- Как исправить: зафиксировать порог в явном виде (`4/6`, `5/6`).

### 9. Ошибка для разбора без образца исправления
- Что не так: материал фиксирует проблему, но не путь коррекции.
- Как исправить: добавить корректное исправление с пояснением.

### 10. Для двух контуров использованы разные базовые запросы
- Что не так: сравнение контуров недостоверно.
- Как исправить: использовать один базовый запрос и фиксировать только целевые изменения итераций.

### 11. Решение по качеству принято без фактов
- Что не так: решение не обосновано.
- Как исправить: в Таблице 2 добавить проверяемые факты по каждому критерию.

### 12. Смешаны разработка материала и отчетность
- Что не так: действия теряют последовательность, растет риск пропуска шага.
- Как исправить: разделить шаги на два режима (`🧱` и `📑`).

## 🧰 Приложение A. Шаблон мини-теста входной проверки (готов к вставке)
### Инструкция
- Вопросов: 6.
- Форматы: один выбор или короткий ответ.
- Время: 8-10 минут.

### Набор вопросов
1. Какой индекс у первого элемента массива?
2. Какой вариант условия цикла корректен для полного прохода по массиву?
3. Что произойдет при обращении к `arr[arr.length]`?
4. В каком случае целесообразно использовать `for`?
5. Найдите ошибку в цикле: `for (i = 0; i <= arr.length; i++)`.
6. Как изменить цикл, чтобы обрабатывались только элементы с нечетными индексами?

### Шкала интерпретации
- `0-2` -> `basic`
- `3-4` -> `standard`
- `5-6` -> `advanced`

### Правильные ответы теста
1. `0`
2. `i < arr.length`
3. Выход за границы массива
4. Когда диапазон индексов известен заранее
5. `<=` заменить на `<`
6. Старт `i = 1`, шаг `i += 2`

### Проходной балл
- Для входной проверки проходной балл не используется как итоговый фильтр; используется интерпретация по диапазонам.

## 📝 Проверка перед фиксацией результата
### Проверка 1. Полнота комплекта
1. В каждом уровне есть теория.
2. В каждом уровне есть 3 задания.
3. В каждом уровне есть ожидаемые ответы и критерии.
4. В каждом уровне есть тест 6 вопросов.
5. В каждом уровне есть правильные ответы теста.
6. В каждом уровне есть проходной балл.
7. В каждом уровне есть ошибка для разбора и образец корректного исправления.

### Проверка 2. Корректность сравнения контуров
1. Один и тот же базовый запрос использован для двух контуров.
2. Есть минимум 2 итерации на контур.
3. В Таблице 2 есть факты по каждому критерию.
4. Решение записано только в Таблице 2.

### Проверка 3. Методическая согласованность
1. Входная проверка действительно влияет на стартовый уровень.
2. Дифференциация отражает разные операции мышления.
3. Уровень `advanced` включает перенос и анализ причины ошибки.
4. Часть B не дублирует часть A.

### Проверка 4. Стилистика и терминология
1. Текст обращен на «вы».
2. Нет лишних упоминаний академического статуса адресата.
3. Текст сохраняет прямое обращение и не уходит в внешнее описание адресата.
4. Термины при первом упоминании даны как «русский + английский в скобках».

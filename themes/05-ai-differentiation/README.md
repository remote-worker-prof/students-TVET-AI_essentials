# Материалы темы `05-ai-differentiation`

> Начинайте с [01_student_start_here.md](01_student_start_here.md).

## Логика темы
Тема 05 посвящена дифференциации обучения с ИИ по формуле:
**единая цель -> диагностика уровня -> разные маршруты -> проверяемый результат**.

Единый мини-кейс темы: `Массивы и циклы`.

## Что обязательно в теме
1. Диагностический вход перед дифференциацией:
   - мини-тест,
   - или teacher-форма в `Yandex Forms`,
   - или оба инструмента сразу.
2. Два ИИ-контура:
   - `local`: `Ollama` или `LM Studio`;
   - `ru-cloud`: `GigaChat` или `YandexGPT`.
3. Выбор рабочего workflow проверки:
   - `Workflow A (Manual Chat)` - обязательный базовый путь;
   - `Workflow B (Automation)` - опциональный расширенный путь.

## Структура
### Основные файлы
- `01_student_start_here.md` - пошаговый старт и маршрут выполнения.
- `02_lecture_differentiation_foundations.md` - основы дифференциации + диагностика уровня.
- `03_lecture_three_level_design.md` - проектирование 3 уровней с конкретными примерами.
- `04_lecture_tool_routes_local_vs_ru_cloud.md` - два workflow: manual и automation.
- `05_practice_track_A_differentiated_module.md` - `A-core`: диагностика, сборка мини-модуля, сравнение local/ru-cloud.
- `06_practice_track_B_variant.md` - `B-variant`: два варианта доработки + опциональная автоматизация.
- `07_debrief_rubric.md` - рубрика и критерии зачета.

### Файлы преподавателя
- `00_teacher_guide.md` - внутренний сценарий и тайминг.
- `08_teacher_opening_script.md` - вводная речь и запуск практики.

## Workflow A: Manual Chat (default)
Применяется как базовый минимум.

Шаги:
1. Провести диагностику уровня.
2. Назначить стартовый уровень.
3. В чатах LLM собрать материалы трех уровней.
4. Сравнить local и ru-cloud по 5 критериям.
5. Принять `approve/revise/reject`.

## Workflow B: Automation (optional)
Опциональный расширенный сценарий на стеке темы 03:
`Yandex Forms -> Albato -> GigaChat -> Google Sheets`.

Вы можете выполнить полный автоматизированный контур, но для зачета достаточно полного `Workflow A`.

## Что вы получаете в результате
- диагностическую карту уровня;
- дифференцированный мини-модуль (3 уровня);
- журнал итераций и таблицу сравнения local/ru-cloud;
- обоснованное решение `approve/revise/reject`;
- готовый комплект для применения на занятии.

## Примечание
Во всех материалах используется академичный формат с обращением на «вы».
Тайминг хранится в `00_teacher_guide.md`.

# Материалы темы `05-ai-differentiation`

> Начинайте с [01_student_start_here.md](01_student_start_here.md).

## Словарь терминов темы
- **Входная проверка подготовки (diagnostic level check)**: короткая проверка, которая помогает выбрать стартовый уровень сложности.
- **Порядок работы (workflow)**: фиксированная последовательность шагов от входной проверки до финального решения.
- **Базовый порядок проверки в чате (Manual Chat Workflow)**: проверка и доработка материалов через чаты языковых моделей.
- **Расширенный автоматизированный порядок проверки (Automation Workflow)**: проверка через связку сервисов `Yandex Forms -> Albato -> GigaChat -> Google Sheets`.
- **Форма преподавателя (teacher form)**: онлайн-форма с вопросами для определения стартового уровня.
- **Обязательная часть A (A-core)**: основной практический маршрут, который выполняется полностью.
- **Вариативная часть B (B-variant)**: дополнительная часть с выбором одного из вариантов.
- **Решение по качеству: принять / доработать / не принимать (approve / revise / reject)**.
- **Локальная модель / российская облачная модель**: два обязательных контура сравнения результатов.

## Логика темы
Тема построена по формуле:
**единая цель -> входная проверка подготовки -> разные уровни заданий -> проверяемый итог**.

Единый мини-кейс: `Массивы и циклы`.

## Что обязательно выполнить
1. Провести входную проверку подготовки:
   - мини-тест,
   - или форма преподавателя в `Yandex Forms`,
   - или оба инструмента.
2. Использовать два контура моделей:
   - локальная модель: `Ollama` или `LM Studio`;
   - российская облачная модель: `GigaChat` или `YandexGPT`.
3. Выполнить обязательную часть A (`A-core`) в [05_practice_track_A_differentiated_module.md](05_practice_track_A_differentiated_module.md).
4. Выполнить один вариант вариативной части B (`B-variant`) в [06_practice_track_B_variant.md](06_practice_track_B_variant.md).
5. Проверить итог по [07_debrief_rubric.md](07_debrief_rubric.md).

## Два порядка работы
- **Базовый порядок проверки в чате (Manual Chat Workflow)**: обязателен и достаточен для зачета.
- **Расширенный автоматизированный порядок проверки (Automation Workflow)**: опционален, дает расширенный практический результат.

# Материалы темы `05-ai-differentiation`

---

## 📚 Словарь терминов
- **Входная проверка подготовки (diagnostic level check)**: короткая проверка, которая помогает выбрать стартовый уровень сложности.
- **Порядок работы (workflow)**: последовательность шагов от входной проверки до итогового решения.
- **Базовый порядок проверки в чате (Manual Chat Workflow)**: создание и доработка материалов через чаты языковых моделей.
- **Расширенный автоматизированный порядок проверки (Automation Workflow)**: маршрут `Yandex Forms -> Albato -> GigaChat -> Google Sheets`.
- **Обязательная часть A (A-core)**: основной практический маршрут темы.
- **Вариативная часть B (B-variant)**: дополнительная доработка после части A.
- **Решение по качеству**: принять / доработать / не принимать (`approve / revise / reject`).

---

## 🎯 Что вы создаете в теме
Вы создаете **три полноценных учебных комплекта** по одной цели:
- уровень `basic`,
- уровень `standard`,
- уровень `advanced`.

Каждый комплект обязательно включает:
1. **Теорию**: краткое объяснение, пример, опорные понятия.
2. **Практику**: 3 задания с ожидаемыми ответами и критериями.
3. **Тест**: 6 вопросов, правильные ответы теста, проходной балл (минимум правильных ответов).
4. **Ошибку для разбора и образец корректного исправления**.

`Ошибка для разбора (в учебном примере)` - это элемент учебного материала, а не статистика ошибок группы.

Единая цель всех уровней: корректно выбирать цикл для прохода по массиву и обнаруживать ошибку индексации.

## 🧱 Разработка материала и 📑 отчётность
- `🧱 Разработка материала`: вы создаете содержание комплектов по уровням.
- `📑 Отчётность`: вы фиксируете результат в таблицах и итоговом комплекте.

В одном подпункте не смешиваются две роли. Сначала создается материал, затем отдельным шагом заполняется отчетная форма.

---

## 🧭 Маршрут выполнения
1. Выполните входную проверку подготовки.
2. Пройдите сквозную лекцию по [09_lecture_integrated_90min.md](09_lecture_integrated_90min.md).
3. Постройте полные комплекты для `basic`, `standard`, `advanced`.
4. Прогоните комплекты через два контура моделей:
   - локальная модель: `Ollama` или `LM Studio`;
   - российская облачная модель: `GigaChat` или `YandexGPT`.
5. Сравните результаты двух контуров и примите итоговое решение по качеству.
6. Выполните один вариант части B без дублирования таблиц части A.

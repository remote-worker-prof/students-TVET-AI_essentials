# 🧪 Входные данные и назначение стартового уровня

---

## 🎯 Зачем нужен этот шаг
Входная проверка подготовки (diagnostic level check) нужна, чтобы подобрать стартовую сложность без перегрузки и без занижения требований [1], [2], [3].

Используются два источника:
1. мини-тест по теме;
2. форма преподавателя (teacher form).

---

## 🧱 Что подготовить перед проверкой
### 🧱 Разработка материала
1. Зафиксировать опорный фрагмент по теме `Массивы и циклы`.
2. Подготовить мини-тест из 6 вопросов.
3. Подготовить форму преподавателя (teacher form) с наблюдаемыми признаками.

### 📑 Отчётность
1. В Таблице 1 оставить строку для мини-теста.
2. В Таблице 1 оставить строку для формы преподавателя.
3. В Таблице 1 оставить строку для итогового уровня.

---

## 🧱 Опорный фрагмент (единый для всех уровней)
1. Массив хранит упорядоченный набор элементов одного типа.
2. Элемент массива выбирается по индексу.
3. Цикл `for` используют, когда число повторений известно заранее.
4. Цикл `while` используют, когда повторение зависит от условия.
5. Ошибка индексации возникает, когда обращение выходит за границы массива.
6. Корректный проход по массиву требует контроля начального и конечного индекса.

---

## 🧪 Мини-тест (6 вопросов)
### 🧱 Разработка материала
Использовать тест как быстрый фильтр стартовой сложности.

1. Что хранит массив?
- A. Набор команд
- B. Упорядоченный набор элементов
- C. Только одну переменную
- D. Только текст

2. С какого индекса обычно начинается массив?
- A. 0
- B. 1
- C. -1
- D. 2

3. Когда обычно выбирают `for`?
- A. Когда число повторений известно
- B. Когда повторений никогда не бывает
- C. Только для строк
- D. Только для ввода с клавиатуры

4. Когда обычно выбирают `while`?
- A. Когда повторение зависит от условия
- B. Когда число повторений заранее известно
- C. Только для массивов из трех элементов
- D. Никогда

5. Что означает ошибка индексации?
- A. Обращение к несуществующему индексу
- B. Ошибка в названии массива
- C. Неверный тип данных
- D. Отсутствие цикла

6. Что помогает избежать выхода за границы массива?
- A. Проверка пределов индекса
- B. Удаление всех условий
- C. Замена массива на строку
- D. Увеличение шага без проверки

### ✅ Правильные ответы теста
`1B, 2A, 3A, 4A, 5A, 6A`

### ✅ Проходные интервалы для назначения уровня
- `0-2` правильных: старт `basic`.
- `3-4` правильных: старт `standard`.
- `5-6` правильных: старт `advanced`.

---

## 📝 Форма преподавателя (teacher form)
### 🧱 Разработка материала
Оценить 5 признаков по шкале `1-3`:
1. Понимание индексации массива.
2. Выбор типа цикла по условию задачи.
3. Самостоятельность объяснения решения.
4. Способность исправить ошибку после подсказки.
5. Перенос решения в измененную задачу.

### ✅ Правила интерпретации суммы
- `5-8` баллов: `basic`.
- `9-12` баллов: `standard`.
- `13-15` баллов: `advanced`.

### 📑 Отчётность
Зафиксировать сумму баллов и уровень по форме в Таблице 1.

---

## 🧭 Алгоритм назначения итогового уровня
### 🧱 Разработка материала
Использовать правило из четырех шагов:
1. Определить уровень по мини-тесту.
2. Определить уровень по форме преподавателя.
3. Сравнить результаты двух источников.
4. Назначить итоговый уровень по таблице решения.

### ✅ Таблица решения
| Ситуация | Итоговый уровень | Пояснение |
|---|---|---|
| Оба источника совпали | Совпавший уровень | Сигнал устойчивый, переход к разработке материалов |
| Разница на 1 уровень | Более низкий из двух | Снижаете риск перегрузки на старте [2] |
| Разница на 2 уровня | `standard` + контрольное задание | После контрольного задания подтвердить или скорректировать уровень |
| Есть только один источник | Уровень по доступному источнику | Зафиксировать ограничение в комментарии |

### 📑 Отчётность
В Таблице 1 обязательно указать:
1. источник,
2. рассчитанный уровень,
3. итоговый уровень,
4. комментарий о причине выбора.

## ✅ Полностью заполненный мини-пример
### 🧱 Разработка материала
- Мини-тест: `4/6` -> `standard`.
- Форма преподавателя: `8/15` -> `basic`.
- Разница на 1 уровень -> итог `basic`.

### 📑 Отчётность
| Блок | Источник/уровень | Цель | Краткое описание уровня | Комментарий |
|---|---|---|---|---|
| входная проверка | mini_test | — | — | `4/6`, уровень `standard` |
| входная проверка | teacher_form | — | — | `8/15`, уровень `basic` |
| входная проверка | final_level | — | — | итог `basic` по правилу разницы в 1 уровень |

---

## ⚠️ Риски шага
1. Решение по уровню принимается «на глаз» без правил.
2. Мини-тест и форма дают разные уровни, но расхождение не объясняется.
3. Итоговый уровень не влияет на глубину заданий в Таблицах 3–5.

---

## 🔧 Как исправить
1. Применять таблицу решения без исключений.
2. Каждое расхождение фиксировать отдельной строкой в Таблице 1.
3. Проверить, что в `advanced` есть перенос и анализ причины ошибки.

---

## 🧩 Расширение: автоматизированный сбор входной проверки
Автоматизация относится к расширенному маршруту и не обязательна.

### 🧱 Разработка материала
1. Создать форму в `Yandex Forms`.
2. Настроить триггер в `Albato` на новую отправку.
3. Передать ответы в `GigaChat` для присвоения уровня и короткого обоснования.
4. Сохранить результат в `Google Sheets`.

### 📑 Отчётность
Зафиксировать ссылку на форму, скрин маппинга полей и строку результата в таблице.

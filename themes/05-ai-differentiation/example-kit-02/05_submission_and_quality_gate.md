# ✅ Сдача и контроль качества блока A

---

## 🎯 Цель документа
Этот документ фиксирует два вопроса:
1. что именно подлежит сдаче;
2. по каким признакам работа считается принятой.

Подход основан на прозрачных критериях и формирующей проверке: порядок проверки результата задан заранее [1], [2], [3].

---

## 📦 Обязательный состав сдачи
1. Таблица 1: входная проверка + матрица уровней.
2. Таблица 2: сравнение двух контуров + решение по качеству.
3. Таблица 3: полный комплект `basic`.
4. Таблица 4: полный комплект `standard`.
5. Таблица 5: полный комплект `advanced`.
6. Финальный комплект материалов в едином виде.

Итоговое решение `принять / доработать / не принимать` допускается только в Таблице 2.

---

## 🧱 Разработка материала и 📑 отчётность: как не смешивать
### 🧱 Разработка материала
Создается учебное содержание:
- теория,
- задания,
- тест,
- правильные ответы теста,
- проходной балл,
- ошибка для разбора и образец корректного исправления.

### 📑 Отчётность
Фиксируется уже созданный результат в таблицах и журнале итераций.

### ⚠️ Неверно
«Сначала заполню таблицу, потом подгоню под нее содержание».

### ✅ Верно
«Сначала собираю содержание, затем документирую в таблице».

---

## 🧪 Качество: минимальные проходные условия
Работа не принимается, если нарушено хотя бы одно условие:
1. отсутствует хотя бы одна из Таблиц 1–5;
2. в Таблицах 3–5 есть пустые обязательные поля;
3. в `advanced` нет задания на перенос и объяснение причины ошибки;
4. нет фактов по критериям в Таблице 2;
5. решение вынесено не в Таблицу 2.

---

## 🧪 Качество: расширенные условия хорошего результата
1. Материалы трех уровней реально различаются по глубине действий.
2. К каждой практике есть ожидаемый ответ и критерий проверки.
3. Для каждого контура есть минимум 2 итерации с наблюдаемым улучшением.
4. Финальный комплект можно применить без дополнительных пояснений.

---

## 🧭 Проверка по шагам перед сдачей
### Шаг 1
- Проверить Таблицу 1: есть входная проверка, есть итоговый уровень, есть матрица.

### Шаг 2
- Проверить Таблицы 3–5: каждый уровень содержит полный комплект.

### Шаг 3
- Проверить Таблицу 2: сравнение двух контуров подтверждено фактами.

### Шаг 4
- Проверить, что итоговое решение записано только в Таблице 2.

### Шаг 5
- Проверить единый финальный комплект: структура понятна и целостна.

---

## 🧩 Минимум и расширение
- **Минимум (обязателен):** базовый порядок проверки в чате (Manual Chat Workflow).
- **Расширение (опционально):** автоматизация `Yandex Forms -> Albato -> GigaChat -> Google Sheets`.

Автоматизация усиливает практику, но не заменяет базовую проверяемость результата [4], [5], [6].

---

## ⚠️ Частые причины возврата на доработку
1. Уровни `basic/standard/advanced` отличаются только длиной текста.
2. Тест есть, но нет правильных ответов теста и проходного балла.
3. «Ошибка для разбора» описана как статистика группы, а не как учебный пример.
4. В Таблице 2 есть баллы, но нет конкретных фактов.

---

## 🔧 Как быстро довести до зачета
1. Для каждого уровня добавить минимум одно уникальное действие, которого нет на соседнем уровне.
2. Привязать каждую задачу к проверяемому критерию.
3. Уточнить «ошибку для разбора» как конкретный фрагмент учебного содержания.
4. Переписать обоснование в Таблице 2 через наблюдаемые факты.

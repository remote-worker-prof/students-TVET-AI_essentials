При проходе по массиву важно выбрать подходящий цикл и не выйти за границы индексов. Цикл `for` удобен, когда известно количество повторений, например число элементов массива. Цикл `while` удобен, когда условие завершения зависит от состояния в процессе выполнения. Базовая ошибка в этой теме - обращение к индексу, которого в массиве нет. Чтобы избежать ошибки, нужно заранее определить начальный индекс и верхнюю границу, а затем сверять условие цикла с длиной массива. На базовом уровне главное - уверенно отличать ситуации для `for` и `while` и проверять границы индекса в каждом примере. |

| Опорные понятия | 3 понятия | `массив`, `индекс`, `границы цикла` |
| Практика 1 | базовое применение + ожидаемый ответ + критерий | Задание: выберите цикл для вывода 5 элементов массива по порядку. Ожидаемый ответ: `for`, потому что число шагов известно заранее. Критерий: назван правильный цикл и дана причина выбора. |
| Практика 2 | поиск ошибки + ожидаемый ответ + критерий | Задание: в записи `for i от 1 до length` найдите ошибку индексации для массива с индексацией с 0. Ожидаемый ответ: начало с 1 пропускает первый элемент, верхняя граница `length` дает выход за массив. Критерий: указаны обе проблемы. |
| Практика 3 | мини-перенос + ожидаемый ответ + критерий | Задание: массив содержит 4 оценки; нужно посчитать сумму всех оценок. Выберите цикл и границы. Ожидаемый ответ: `for i от 0 до 3` или условие `i < length`. Критерий: корректно заданы начальная и конечная границы. |
| Тест | 6 вопросов | 1) Что хранит массив? A команды, B упорядоченные элементы, C один символ, D один цикл.<br>2) Какой стартовый индекс обычно используется? A 0, B 1, C -1, D 2.<br>3) Когда удобнее `for`? A заранее известно число шагов, B шаги неизвестны и нет условия, C только для строк, D никогда.<br>4) Когда удобнее `while`? A когда есть условие продолжения, B только при фиксированной длине, C только для массивов из 2 элементов, D только для сортировки.<br>5) Что такое выход за границы массива? A обращение к несуществующему индексу, B переименование массива, C смена типа, D ошибка ввода.<br>6) Что предотвращает ошибку индексации? A проверка условия `i < length`, B случайный шаг, C удаление условия, D удвоение индекса. |

| Образец корректного исправления | 1 корректный способ исправления | Исправление: заменить условие на `i < length` (или верхнюю границу `length-1` при включенной правой границе). |

## 📑 Таблица 4. Полный комплект уровня `standard`

|---|---|---|
| Теория | 120-180 слов, акцент на самостоятельный выбор | На уровне `standard` основной акцент переносится с распознавания шаблона на самостоятельный выбор стратегии прохода по массиву. Вы выбираете `for`, если число повторений заранее определено длиной массива. Вы выбираете `while`, если повторение зависит от условия, которое может измениться в процессе обработки данных. При этом одно и то же действие может быть реализовано разными циклами, но корректность определяется не формой записи, а условиями остановки и контролем границ индекса. Ожидаемый ответ: двойное увеличение индекса нарушает последовательный проход; часть элементов пропускается, возможен выход за пределы при последней итерации. 4. Итоговое решение по качеству зафиксировано в Таблице 2.

---

## 📝 Проверка перед отправкой
1. Таблицы 1–5 заполнены без пустых обязательных полей.
2. Для каждого контура есть минимум 2 итерации с фактами улучшений.
3. Решение указано только в Таблице 2.
4. В `advanced` есть перенос и анализ причины ошибки.
